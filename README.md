# Endterm_Grupo_07

Ejercicio #1 Desarrollo de un vehículo.<br/>

El objetivo de este ejercicio es desarrollar de forma colaborativa una clase Vehículo utilizando Test
Driven Development (TDD), git, GitHub y JUnit. Al finalizar, el repositorio debe reflejar buenas prácticas
de control de versiones, tests automatizados y colaboración en equipo.

- Practicar con el sistema de Control de Versiones git 
- Preparación del repositorio en Github con protección de la rama de producción, entre otras
  cosas. 
- Test Driven Development (TDD):
  - Elaboración de test cases
  - Programación de tests unitarios con Junit
  - Desarrollo de la aplicación
- Revisión colaborativa de código en Github para la detección de bugs y mejoras
- Automatización de la ejecución de los tests en Github
- El repositorio resultante debe reflejar con commits una repartición equitativa del trabajo
